⚙️ How Idempotency Works

STEP 1
First Payment
Sent with a unique key. Server processes payment and saves result in Redis for 60s.
STEP 2
Network Drops
Toggle the network switch above to simulate a connection failure mid-payment.
STEP 3
User Retries
Click "Retry Payment". Same Idempotency-Key is sent again.
STEP 4
Safe Response
Server detects duplicate key → returns cached result. No double payment!
